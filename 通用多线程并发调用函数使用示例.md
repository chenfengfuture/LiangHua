# 通用多线程并发调用函数使用示例

## 函数概述

在 `StockBasicService` 类中新增了 `execute_concurrent_tasks` 函数，这是一个通用的多线程并发调用函数，能够：

1. 接收一个基础函数（如获取上交所股票数据的函数）和一组参数列表
2. 利用现有的 `thread_pool.py` 中的 `run_concurrent_tasks` 函数并发执行多个任务
3. 合并所有任务的返回结果（例如将所有页的股票列表合并为一个完整列表）
4. 处理异常：单个任务失败不应影响其他任务，最终返回成功和失败的任务信息

## 函数签名

```python
def execute_concurrent_tasks(self, base_func: Callable, param_list: List[Dict[str, Any]], 
                            max_workers: int = 5, timeout: int = 60) -> Dict[str, Any]:
```

### 参数说明

- **base_func**: 基础函数，需要并发执行的函数
- **param_list**: 参数列表，每个元素为字典，包含：
  - `args`: 位置参数元组（可选）
  - `kwargs`: 关键字参数字典（可选）
  - `name`: 任务名称（可选，默认使用函数名+索引）
- **max_workers**: 最大并发线程数，默认5
- **timeout**: 整体超时时间（秒），默认60

### 返回值格式

```python
{
    "success": bool,      # 整体调用是否成功（至少有一个任务成功）
    "data": {
        "all_results": list,     # 所有任务的原始结果
        "success_results": list, # 成功的任务数据
        "failed_results": list,  # 失败的任务信息
        "merged_data": any,      # 合并后的数据（根据任务类型）
        "statistics": {          # 统计信息
            "total_tasks": int,
            "success_count": int,
            "fail_count": int,
            "success_rate": float
        }
    },
    "message": str        # 成功或错误信息（包含统计信息）
}
```

## 使用示例

### 示例1：分页查询股票列表

```python
from backend.stock_services.services.stock_basic import StockBasicService

# 创建服务实例
service = StockBasicService()

# 假设有一个分页查询股票列表的函数
def get_stock_list_by_page(page: int, page_size: int = 100):
    """模拟分页查询股票列表"""
    # 这里调用实际的API
    return [f"股票_{page}_{i}" for i in range(page_size)]

# 准备参数列表（查询1-5页的数据）
param_list = [
    {"args": (1, 100), "name": "page_1"},
    {"args": (2, 100), "name": "page_2"},
    {"args": (3, 100), "name": "page_3"},
    {"args": (4, 100), "name": "page_4"},
    {"args": (5, 100), "name": "page_5"},
]

# 执行并发任务
result = service.execute_concurrent_tasks(
    base_func=get_stock_list_by_page,
    param_list=param_list,
    max_workers=3,  # 最多3个线程并发
    timeout=30      # 整体超时30秒
)

if result["success"]:
    data = result["data"]
    print(f"成功获取 {len(data['merged_data'])} 条股票数据")
    print(f"成功率: {data['statistics']['success_rate']:.2%}")
else:
    print(f"任务失败: {result['message']}")
```

### 示例2：批量查询个股信息

```python
from backend.stock_services.services.stock_basic import StockBasicService

# 创建服务实例
service = StockBasicService()

# 使用现有的 get_stock_individual 函数
# 准备要查询的股票代码列表
symbols = ["SH601127", "SZ000001", "SH600519", "SZ000858", "SH601318"]

# 构建参数列表
param_list = []
for symbol in symbols:
    param_list.append({
        "args": (symbol,),
        "name": f"query_{symbol}"
    })

# 执行并发任务
result = service.execute_concurrent_tasks(
    base_func=service.get_stock_individual,
    param_list=param_list,
    max_workers=2,  # 限制并发数，避免API限制
    timeout=60
)

if result["success"]:
    data = result["data"]
    print(f"成功查询 {data['statistics']['success_count']} 只股票")
    
    # 处理成功的结果
    for task_result in data["success_results"]:
        print(f"{task_result['name']}: 查询成功")
    
    # 处理失败的结果
    if data["failed_results"]:
        print(f"失败任务:")
        for failed_task in data["failed_results"]:
            print(f"  - {failed_task['name']}: {failed_task['error']}")
```

### 示例3：混合参数类型（位置参数和关键字参数）

```python
from backend.stock_services.services.stock_basic import StockBasicService

# 创建服务实例
service = StockBasicService()

        "period": f"{start_date} 到 {end_date}" if start_date and end_date else "最近"
    }

# 执行并发任务
result = service.execute_concurrent_tasks(
    base_func=query_stock_data,
```

### 示例4：处理API接口分页

```python
from backend.stock_services.services.stock_basic import StockBasicService
import time

# 创建服务实例
service = StockBasicService()

def fetch_api_data(api_url: str, params: dict, retry_count: int = 3):
    """模拟API调用，支持重试"""
    # 这里调用实际的API
    time.sleep(0.5)  # 模拟网络延迟
    return {
        "url": api_url,
        "params": params,
        "data": f"API返回的数据: {params}"
    }

# 模拟分页API调用
base_url = "https://api.example.com/stocks"
param_list = []

# 生成10页的请求参数
for page in range(1, 11):
    param_list.append({
        "args": (base_url,),
        "kwargs": {
            "params": {"page": page, "page_size": 100},
            "retry_count": 2
        },
        "name": f"api_page_{page}"
    })

# 执行并发任务（限制并发数避免被API限制）
result = service.execute_concurrent_tasks(
    base_func=fetch_api_data,
    param_list=param_list,
    max_workers=5,  # 最多5个并发请求
    timeout=120     # 整体超时2分钟
)

if result["success"]:
    data = result["data"]
    print(f"API调用统计:")
    print(f"  总请求数: {data['statistics']['total_tasks']}")
    print(f"  成功数: {data['statistics']['success_count']}")
    print(f"  失败数: {data['statistics']['fail_count']}")
    print(f"  成功率: {data['statistics']['success_rate']:.2%}")
    
    # 提取所有成功的数据
    all_data = []
    for task_result in data["success_results"]:
        all_data.append(task_result["data"])
    
    print(f"总共获取 {len(all_data)} 页数据")
```

## 数据合并策略

函数内置了智能数据合并策略，根据返回数据的类型自动选择合并方式：

### 1. 列表数据合并
如果所有任务返回的都是列表，则合并为一个列表：

```python
# 任务1返回: [1, 2, 3]
# 任务2返回: [4, 5, 6]
# 合并结果: [1, 2, 3, 4, 5, 6]
```

### 2. 字典数据合并
如果所有任务返回的都是字典，则合并为一个字典：

```python
# 任务1返回: {"a": 1, "b": 2}
# 任务2返回: {"c": 3, "d": 4}
# 合并结果: {"a": 1, "b": 2, "c": 3, "d": 4}
# 注意：相同键会被后面的值覆盖
```

### 3. 混合数据类型
如果数据类型不一致，则返回原始结果列表：

```python
# 任务1返回: [1, 2, 3]
# 任务2返回: {"a": 1}
# 合并结果: [[1, 2, 3], {"a": 1}]
```

## 错误处理

函数具有完善的错误处理机制：

1. **参数验证**：验证基础函数是否可调用，参数列表是否有效
2. **任务级容错**：单个任务失败不影响其他任务
3. **异常捕获**：捕获所有异常，返回详细的错误信息
4. **超时控制**：支持整体超时控制，防止任务无限等待
5. **结果统计**：提供详细的成功/失败统计信息

## 最佳实践

1. **合理设置并发数**：根据API限制和系统资源设置合适的 `max_workers`
2. **设置超时时间**：根据任务复杂度设置合理的 `timeout`
3. **使用任务名称**：为每个任务指定有意义的名称，便于调试
4. **处理失败任务**：总是检查 `failed_results`，了解失败原因
5. **验证返回数据**：使用 `merged_data` 或分别处理 `success_results`


## 性能考虑

1. **内存使用**：大量任务并发时注意内存使用
2. **API限制**：遵守第三方API的调用频率限制
3. **连接池**：函数内部使用线程池，避免频繁创建销毁线程
4. **超时设置**：合理设置超时，避免资源浪费

`execute_concurrent_tasks` 函数提供了一个通用、灵活、健壮的多线程并发调用解决方案，特别适合：

- 分页数据获取
- 批量查询任务
- API接口并发调用
- 数据处理任务并行化

通过合理使用该函数，可以显著提高数据获取效率，同时保持代码的简洁性和可维护性。
