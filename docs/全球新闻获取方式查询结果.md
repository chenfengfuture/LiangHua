## 四、数据限制

### 1. 采集频率限制

- **最小间隔**：900 秒 = 15 分钟
- **原因**：遵循东方财富/同花顺接口反爬虫策略，避免频繁请求被封禁
- **控制机制**：通过 Redis 记录上次采集时间，`elapsed < 900` 则跳过本次采集

### 2. 数据范围限制

- **增量采集**：只采集**上次采集时间**到**当前时间**之间的增量数据
- **去重机制**：基于标题去重，同一标题只采集一次
- **内存过滤**：获取全量数据后在内存中按时间范围过滤，只保留增量数据

### 3. 接口限制

- AkShare 接口 `stock_info_global_ths()` 返回**最新新闻**，不支持指定时间范围参数
- 每次获取全部最新新闻，系统在内存中过滤出增量数据
- 接口不提供原始新闻链接 URL

### 4. LLM 分析限制

- 默认所有全球新闻都进入 LLM 分析队列（除非敏感内容被过滤）
- 敏感词过滤规则与其他板块一致，包含政治/高风险词汇不进入 LLM

## 五、存储架构

### 1. MySQL 分表存储

- **分表规则**：按年月分表，表名格式 `news_global_YYYYMM`
- **确保表存在**：采集前自动调用 `ensure_table_exists("global", year_month)` 创建表
- **表结构**：与其他新闻板块一致，包含所有分析字段（12个AI字段可为空）

### 2. Redis 三层管道（与其他新闻板块一致）

| Redis Layer | Key 格式 | 数据结构 | 作用 |
|-------------|----------|----------|------|
| Layer 1 | `news:data:{id}` | String | 存储完整新闻 JSON |
| Layer 2 | `news:pending_llm` | Set | 待 LLM 分析的新闻 id 集合 |
| Layer 3 | `news:pending_persist` | List | 已分析待持久化的新闻 id 列表 |

## 六、调度机制

### 1. 定时调度

- **调度入口**：`collect_all_news()` 函数

### 2. 频率控制代码

```python

if elapsed is not None and elapsed < 900:
    print(f"[global] 跳过: 距上次采集 {elapsed:.0f}s < 900s (15分钟)")
```

## 七、异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| AkShare 接口调用失败 | 重试 1 次，仍失败返回 0 条，不影响其他板块采集 |
| 获取到空数据 | 记录日志，更新采集时间，返回 0 条 |
| 解析时间失败 | 跳过该条新闻，继续处理其他 |
| 标题重复 | 跳过重复标题 |
| 时间不在增量范围内 | 跳过该条 |
| 整个采集过程异常 | 捕获异常，打印错误信息，返回 0 条，不崩溃 |
