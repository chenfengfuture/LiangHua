# 项目量化新闻系统 - 完整技术文档

## 1. 项目整体介绍

### 项目定位
量华量化新闻系统是一个集新闻实时采集、AI智能分析、实时推送于一体的量化交易辅助平台。系统专注于财经新闻的自动化处理，为投资决策提供数据支持。

### 核心功能
1. **新闻实时采集**：多源新闻并行采集，支持增量更新
2. **新闻全量补全**：历史数据批量拉取与去重处理
3. **LLM自动分析**：AI新闻内容深度解析与情感分析
4. **WebSocket实时推送**：实时新闻更新推送至前端展示

### 技术栈
- **后端**：Python + FastAPI + MySQL + Redis
- **数据源**：AKShare / QStock / 东方财富API / 财联社API
- **AI模型**：火山方舟大模型 + FinBERT情感分析
- **前端**：React + TypeScript + WebSocket
- **部署**：Windows服务 + 自动化定时任务

### 架构模式
- 前后端分离架构
- 微服务化设计
- 异步任务处理
- 三层Redis管道架构

## 4. 新闻模块完整逻辑

### 4.1 实时新闻采集
- **数据源**：akshare.py 多接口定时拉取
- **采集频率**：每1分钟触发一次全量采集
- **采集板块**：5大板块并行采集
- **增量机制**：基于Redis记录上次采集时间

### 4.2 全量新闻补全
- **数据源**：qstock按日期范围批量拉取
- **去重规则**：content_hash = md5(title + content)
- **入库策略**：不存在则插入，存在则跳过
- **批量处理**：500条/批次写入

### 4.3 数据去重规则
```python
content_hash = hashlib.md5(f"{title}{content}".encode("utf-8")).hexdigest()
```
- 唯一索引：`UNIQUE INDEX uk_content_hash (content_hash)`
- 去重判断：入库前检查content_hash是否存在
- 容错机制：INSERT IGNORE + 批量回查

### 4.4 入库规则
1. **新数据**：计算content_hash → 检查重复 → 插入MySQL → 获取自增id
2. **重复数据**：直接跳过，不重复插入
3. **批量优化**：500条/批次，减少数据库连接开销

### 4.5 分析规则
```python
need_analyze = 1  # 进入LLM分析队列
need_analyze = 0  # 已分析完成或跳过
```
- **分析条件**：新闻类型+内容敏感性判断
- **CCTV过滤**：只分析产业政策相关新闻
- **敏感词过滤**：政治、风险等敏感内容跳过分析

### 4.6 推送规则
1. **实时推送**：采集完成立即通过WebSocket推送原始新闻
2. **分析推送**：LLM分析完成推送分析结果
3. **订阅机制**：前端可按频道订阅不同类型新闻

### 4.7 数据流向
```
采集层 → MySQL入库 → Redis缓存层 → LLM分析层 → 持久化层 → 前端展示
     ↓           ↓            ↓           ↓            ↓
原始新闻    自增ID获取   待分析队列    批量分析    结果更新   实时推送
```

## 5. akshare.py所有接口明细

### 5.1 5大新闻采集板块

#### 板块1：个股/公司新闻 (company)
- **函数名称**：`_fetch_company()`
- **接口作用**：采集个股和公司相关新闻
- **数据源**：`ak.stock_news_em()`
- **目标股票**：`["300059", "600519", "300750", "600030", "688981"]`
- **采集频率**：每小时最多一次
- **返回字段**：
  - `title`: 新闻标题
  - `content`: 新闻内容
  - `url`: 原文链接
  - `source`: 来源媒体（东方财富）
  - `source_category`: 来源分类（东方财富）
  - `news_type`: 新闻类型（company）
  - `publish_time`: 发布时间

#### 板块2：财联社新闻 (cls)
- **函数名称**：`_fetch_cls()`
- **接口作用**：采集财联社财经新闻
- **数据源**：`ak.stock_info_global_cls()`
- **采集频率**：每小时最多一次
- **返回字段**：
  - `title`: 新闻标题
  - `content`: 新闻内容
  - `url`: None（无原文链接）
  - `source`: 财联社
  - `source_category`: 财联社
  - `news_type`: 新闻类型（cls）
  - `publish_time`: 发布时间

#### 板块3：全球新闻 (global)
- **函数名称**：`_fetch_global()`
- **接口作用**：采集全球财经新闻
- **数据源**：`ak.stock_info_global_ths()`
- **采集频率**：每15分钟最多一次
- **返回字段**：
  - `title`: 新闻标题
  - `content`: 新闻内容
  - `url`: None（无原文链接）
  - `source`: 东方财富
  - `source_category`: 东方财富全球
  - `news_type`: 新闻类型（global）
  - `publish_time`: 发布时间

#### 板块4：新闻联播 (cctv)
- **函数名称**：`_fetch_cctv()`
- **接口作用**：采集新闻联播内容
- **数据源**：`ak.news_cctv()`
- **采集时间**：每天19:30后执行一次
- **去重机制**：Redis标记当日已采集
- **内容过滤**：只保留产业政策相关新闻
- **返回字段**：
  - `title`: 新闻标题
  - `content`: 新闻内容
  - `url`: None（无原文链接）
  - `source`: 新闻联播
  - `source_category`: 新闻联播
  - `news_type`: 新闻类型（cctv）
  - `publish_time`: 发布时间（默认19:00）

#### 板块5：研报新闻 (report)
- **函数名称**：`_fetch_report()`
- **接口作用**：采集当天深度研报
- **数据源**：`ak.stock_research_report_em()`
- **采集规则**：只采集当天发布的研报
- **返回字段**：
  - `title`: 报告名称
  - `content`: 机构+评级+行业信息
  - `url`: None（无原文链接）
  - `source`: 东方财富
  - `source_category`: 东方财富研报
  - `news_type`: 新闻类型（report）
  - `publish_time`: 发布时间

## 6. 其他核心函数参数与返回值

### 6.1 LLM新闻分析函数

#### 函数名称：`LLM.chat()`
- **功能说明**：调用大模型进行新闻分析
- **所属模块**：`utils/llm.py`
- **输入参数**：
  - `prompt`: 分析提示词（系统预设）
  - `model`: 模型名称（默认：deepseek-v3.2）
  - `system_prompt`: 系统提示词（可选）
  - `images`: 图片列表（支持URL/本地路径/Base64）
  - `files`: 文件列表（支持多种格式）
- **输出参数**：
  - `ai_interpretation`: AI核心解读（200字要点概括）
  - `ai_event_type`: 事件类型枚举（财报/并购/政策/研发等）
  - `ai_impact_level`: 影响等级（1-5级）
  - `ai_impact_direction`: 影响方向（1=利好，0=中性，-1=利空）
  - `ai_risk_level`: 风险等级（1-5级）
  - `ai_benefit_sectors`: 受益板块（中文逗号分隔）
  - `ai_benefit_stocks`: 受益个股（中文逗号分隔）
  - `ai_keywords`: 核心关键词（3-6个，逗号分隔）
  - `sentiment`: 全文情感得分（-1.0 ~ 1.0）
  - `sentiment_label`: 情感标签（1=正面，0=中性，-1=负面）

#### 函数名称：`LLM.stream()`
- **功能说明**：流式调用大模型
- **输入参数**：同`chat()`函数
- **输出**：生成器，实时返回分析结果片段

#### 函数名称：`LLM.chat_with_history()`
- **功能说明**：带历史上下文的对话
- **输入参数**：
  - `prompt`: 当前提示词
  - `history`: 历史对话列表
  - 其他参数同`chat()`
- **输出**：完整分析结果

### 6.2 数据库入库函数

#### 函数名称：`insert_news()`

  1. 计算content_hash
  2. 检查数据库是否已存在
  3. 不存在则写入MySQL
  4. 获取自增id返回
