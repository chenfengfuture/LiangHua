# 全量新闻获取函数升级说明

## 升级概述

已成功升级 `backend/utils/full_news_get.py` 的全量获取函数，支持手动/自动双模式。

## 核心功能

### 1. 手动/自动双模式 (`is_manual` 参数)

- **手动模式** (`is_manual=True`): API调用时使用
  - 获取新闻 → 清洗 → 入库 → Redis → LLM队列 → **WebSocket推送前端**
  - 完全遵循现有的实时新闻获取-入库-推送完整流程

- **自动模式** (`is_manual=False`): 项目启动时使用
  - 获取新闻 → 清洗 → 入库 → Redis → LLM队列 (**不推送WebSocket**)
  - 静默执行，不干扰前端

### 2. 可用函数

```python
# 1. 日期范围补全（支持手动/自动模式）
def repair_global_news_date_range(
    start_date: str,          # 开始日期，格式 YYYY-MM-DD
    end_date: str = None,     # 结束日期，默认与开始日期相同
    is_manual: bool = False   # 模式：True=手动，False=自动
) -> Dict

# 2. 单日补全（支持手动/自动模式）
def repair_global_news_single_date(
    date: str,                # 日期，格式 YYYY-MM-DD
    is_manual: bool = False   # 模式：True=手动，False=自动
) -> Dict

# 3. 兼容性函数（保持与原有接口一致）
def repair_daily_news_func(
    start_date: str,
    end_date: str = None,
    is_manual: bool = False
) -> Dict
```

### 3. 返回结果格式

```python
{
    "success": True/False,           # 是否成功
    "start_date": "2026-04-01",      # 开始日期
    "end_date": "2026-04-03",        # 结束日期
    "total_days": 3,                 # 总天数
    "total_fetched": 150,            # 获取的新闻条数
    "total_pushed": 145,             # 成功推入系统的条数
    "message": "成功获取 150 条，推入系统 145 条",  # 状态消息
    "error": "错误信息"               # 失败时的错误信息（仅当success=False时）
}
```

## 使用示例

### 示例 1: 手动模式（API调用）

```python
from utils.full_news_get import repair_global_news_date_range

# 手动补全2026年4月1日到3日的新闻
result = repair_global_news_date_range(
    start_date="2026-04-01",
    end_date="2026-04-03",
    is_manual=True  # 手动模式，会推送WebSocket到前端
)

if result["success"]:
    print(f"成功补全 {result['total_pushed']} 条新闻")
    print(f"WebSocket已推送到前端")
```

### 示例 2: 自动模式（项目启动）

```python
from utils.full_news_get import repair_global_news_single_date

# 自动补全昨天的新闻（项目启动时调用）
result = repair_global_news_single_date(
    date="2026-04-10",
    is_manual=False  # 自动模式，不推送WebSocket
)

if result["success"]:
    print(f"静默补全 {result['total_pushed']} 条新闻")
    # 前端不会收到推送通知
```

### 示例 3: 通过API调用

```python
# 调用 routes.py 中的 repair_daily_news API
# POST /api/news/daily_news/repair?start_date=2026-04-01&end_date=2026-04-03

# 这会调用 repair_daily_news_func，默认使用手动模式
# 结果会通过WebSocket推送到前端
```

## 技术实现细节

### 1. 数据处理流程

```
手动模式 (is_manual=True):
  获取新闻 → 清洗 → 入库 → Redis缓存 → LLM分析队列 → WebSocket推送前端

自动模式 (is_manual=False):
  获取新闻 → 清洗 → 入库 → Redis缓存 → LLM分析队列
  （不推送WebSocket）
```

### 2. 与现有系统的兼容性

- ✅ **不修改现有实时新闻代码**：完全独立封装
- ✅ **复用现有逻辑**：去重、清洗、入库、Redis、LLM全部沿用现有逻辑
- ✅ **时间字段独立**：全量补全时间记录独立，不影响实时采集时间
- ✅ **API兼容**：保持与 `routes.py` 中 `repair_daily_news` 的兼容性

### 3. 错误处理与稳定性

- ✅ **网络请求重试**：所有网络请求都有完整 try-except
- ✅ **数据验证**：时间解析失败跳过单条，不中断流程
- ✅ **分页控制**：最多25页限制，防止死循环
- ✅ **去重机制**：基于 `content_hash` 的全局去重
- ✅ **异常捕获**：所有函数都有完整异常捕获

## 部署说明

### 1. 项目启动时自动补全

在项目启动脚本中添加：

```python
# 项目启动时自动补全最近3天的新闻
from utils.full_news_get import repair_global_news_date_range
from datetime import datetime, timedelta

# 计算最近3天
end_date = datetime.now().strftime("%Y-%m-%d")
start_date = (datetime.now() - timedelta(days=3)).strftime("%Y-%m-%d")

# 自动模式补全
result = repair_global_news_date_range(
    start_date=start_date,
    end_date=end_date,
    is_manual=False  # 自动模式，不干扰前端
)
```

### 2. 手动触发补全

通过API调用手动触发：

```bash
# 调用 repair_daily_news API
curl -X POST "http://localhost:8001/api/news/daily_news/repair?start_date=2026-04-01&end_date=2026-04-03"
```

## 注意事项

1. **网络依赖**：需要稳定的网络连接访问东方财富API
2. **性能优化**：批量处理，减少数据库和Redis操作
3. **错误恢复**：支持断点续传，重复执行同一日期范围只会插入新数据
4. **资源占用**：自动模式在项目启动时执行，避免高峰时段
5. **日志记录**：所有操作都有详细日志，便于排查问题
