## 异常处理流程

```
用户请求 → 路由层 → 服务层(stock_service.py)
                     ↓
异常抛出 → 全局异常处理器 → 日志记录 → 统一错误响应 → 返回用户
```

### 支持的异常类型：
1. **`ValidationException`** - 参数验证失败 (HTTP 422)
2. **`DataNotFoundError`** - 数据不存在 (HTTP 404)
3. **`DatabaseException`** - 数据库操作失败 (HTTP 503)
4. **`ExternalServiceException`** - 外部服务调用失败 (HTTP 502)

## 日志记录机制

### 统一日志配置
- 使用 `config/logging_config.py` 的 `log_exception()` 函数
- 支持不同日志级别（error, warning, info, debug）
- 包含服务名称和异常上下文

### 日志内容：
- 异常类型和消息
- 服务名称和操作上下文
- 异常堆栈信息（开发环境）
- 请求上下文（URL、方法、IP）

## 使用示例

### 1. 基本使用
```python
from stock_services.services.commont.stock_service import BaseStockService

service = BaseStockService(service_name="MyStockService")

try:
    # 获取股票基本信息
    info = service.get_stock_basic_info("000001")
    print(f"股票信息: {info}")
    
except ValidationException as e:
    print(f"参数错误: {e}")
except DataNotFoundError as e:
    print(f"数据不存在: {e}")
except DatabaseException as e:
    print(f"数据库错误: {e}")
except ExternalServiceException as e:
    print(f"外部服务错误: {e}")
```

### 2. 搜索股票
```python
try:
    # 搜索股票
    results = service.search_stock("平安", "A股")
    print(f"找到 {results['count']} 只股票")
    
except Exception as e:
    # 所有异常都会被全局异常处理器捕获
    print(f"搜索失败: {e}")
```

## 集成说明

### 1. 与全局异常处理器集成
所有从服务层抛出的异常都会被 `main.py` 中注册的全局异常处理器捕获，转换为标准化的错误响应格式。

### 2. 与日志系统集成
使用统一的日志配置系统，确保所有异常都被正确记录，便于问题排查和系统监控。

### 3. 与现有代码兼容
- 不修改现有接口签名
- 保持向后兼容性
- 逐步替换原有异常处理逻辑

## 性能考虑

### 异常处理开销
- 异常对象的创建和抛出有轻微性能开销
- 在正常流程中应避免不必要的异常抛出
- 数据不存在等常见情况应返回 `None` 而不是抛出异常

### 日志记录优化
- 生产环境减少详细日志输出
- 开发环境保留完整堆栈信息
- 支持日志轮转和分级存储
